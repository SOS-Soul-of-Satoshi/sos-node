# Soul of Satoshi Support

This repository is the durable support record for the SOS public testnet.

## Where To Ask

- Community chat and live network status: [SOS Discord](https://discord.gg/dbbjsFBchP)
- Durable questions, setup help, and general feedback: [GitHub Discussions](https://github.com/SOS-Soul-of-Satoshi/sos-node/discussions)
- Reproducible software defects: [bug report](https://github.com/SOS-Soul-of-Satoshi/sos-node/issues/new?template=bug_report.yml)
- Validator onboarding or operations: [validator support](https://github.com/SOS-Soul-of-Satoshi/sos-node/issues/new?template=validator_support.yml)
- Product or protocol proposals: [feature request](https://github.com/SOS-Soul-of-Satoshi/sos-node/issues/new?template=feature_request.yml)
- Suspected vulnerabilities: [private security advisory](https://github.com/SOS-Soul-of-Satoshi/sos-node/security/advisories/new)

Search existing issues and Discussions before opening a new report. Include the
release version, operating system, exact command, observed result, expected
result, and redacted logs where relevant.

## Secret Safety

Never post seed phrases, private keys, API tokens, cookies, passwords, complete
environment files, keystores, or unredacted authentication headers. Testnet
tokens have no monetary value, but test credentials can still expose hosts and
accounts. If a report contains a secret, revoke it before doing anything else.

Security reports do not belong in Issues or Discussions. Use the private
advisory route in [SECURITY.md](SECURITY.md).

## Scope

The public testnet is pre-mainnet, unaudited software. Support is best effort.
The public faucet is for ordinary wallet testing and cannot fund the validator
self-bond. Validator operators should follow the onboarding documentation in
the current release and use the validator support form for deployment evidence.
